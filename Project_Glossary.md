# SOP Glossary

## To ensure a smooth onboarding experience, this project uses specific terms to describe the workflow and tasks. If you're new here, this guide will help you get up to speed.

- Beginner-Friendly Issue: A task identified by a maintainer as being small, low-risk, and suitable for a first-time contributor.

- BPMN (Business Process Model and Notation): A graphical representation used to specify and model business processes in a workflow.

- Camunda Modeler	A desktop or web application used to design and edit BPMN diagrams and process maps.

- Definition of Success: A clear statement in an issue description that outlines exactly what a completed task should look like.

- Discoverability: The ease with which a new contributor can find a task, which is improved by applying specific labels.

- Good First Issue: A specific label applied to GitHub issues to signal that the task is a welcoming entry point for newcomers.

- Help Wanted: A label used to indicate that a project is actively seeking contributors for a specific task.

- Issue Template: A pre-configured form in GitHub that provides a structure for reporting bugs or onboarding tasks.

- Maintainer: An individual responsible for identifying tasks, drafting documentation, and managing the project's issues.

- Open Source: A type of project where the original source code is made freely available for others to contribute to and modify.

- Unassigned: The status of an issue without a designated worker; onboarding issues are left unassigned so contributors can express their own interest.
